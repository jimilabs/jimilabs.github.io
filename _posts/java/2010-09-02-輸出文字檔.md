在對編碼有初步了解後，來看一個跟編碼有相關的Java應用：輸出文字檔

我們使用OutputStreamWriter來實作, 先來看第一種做法:

{% highlight java %}
public static void test1() {
  try {
    String filePath = "test.txt"; // 檔案路徑
    FileOutputStream fileOutputStream = new FileOutputStream(filePath);
    OutputStreamWriter outputStreamWriter = new OutputStreamWriter(fileOutputStream);
    outputStreamWriter.write("測試");
    outputStreamWriter.close();
  } catch (Exception e) {
    e.printStackTrace();
  }
}			
{% endhighlight %} 		

	
OutputStreamWriter會使用系統預設的編碼，因此用這一種寫法所輸出的文字檔會因為作業系統不同而使用不同的編碼格式輸出，也就很容易出現一種問題，就是在本機開發程式時，測試結果都是正常的，只是一旦部署到正式機時，就出現亂碼的問題了．

所以，我們再來看看第二種指定編碼的寫法：

{% highlight java %}
    public static void test2() {
        try {
            String filePath = "test.txt"; // 檔案路徑
            FileOutputStream fileOutputStream = new FileOutputStream(filePath);
            OutputStreamWriter outputStreamWriter = new OutputStreamWriter(fileOutputStream, "big5");
      outputStreamWriter.write("測試");
      outputStreamWriter.close();
    } catch (Exception e) {
      e.printStackTrace();
    }
  }					
{% endhighlight %} 
			
在上面那段程式中，指定了輸出為big5編碼的文字檔，這樣就能解決在不同的環境下，輸出文字檔編碼不同的問題了．不過，還是有一種情況要注意，請看下面的例子：

{% highlight java %}
  public static void test2() {
    try {
      String filePath = "test.txt"; // 檔案路徑
      FileOutputStream fileOutputStream = new FileOutputStream(filePath);
      OutputStreamWriter outputStreamWriter = new OutputStreamWriter(fileOutputStream, "UTF16");
      outputStreamWriter.write("測試");
      outputStreamWriter.close();
    } catch (Exception e) {
      e.printStackTrace();
    }
  }									
{% endhighlight %} 
			
這種寫法輸出文字檔的編碼是UTF-16 BE, 為了使程式碼更清析易讀, 在宣告UTF16的編碼時,可明確指定為UTF-16LE 或是 UTF-16BE．
