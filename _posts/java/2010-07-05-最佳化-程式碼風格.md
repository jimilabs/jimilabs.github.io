### 基本語法


#### 刪除空白的函式

有些工具能為我們產生空白的函式，再由程式設計師填入函式內的實際作用程式碼，不過，在實際開發時才會發現並不是每個函式都用得到，因此就會留下一些空的函式．這些空的函式，在最後編譯出的bytecode中，還是會出現，也會被呼叫．但他們實際上並沒有用途，因此我們可以放心移除它們，以換取更高效能的程式碼．

 
#### 刪除無用的程式碼

一份程式碼，在多個程式設計師的修改後，或許是邏輯改變了，也可能是因為粗心大意，而產生一些永遠不會被執行到的程式碼．刪除這些程式碼可以讓程式更易讀，更快速．


#### 展開迴圈

有時候，使用迴圈並不是必要的，如底下的這個例子：

    int[] x = new int[5];
    for (int i=0; i<5; i++) {
        x[i] = 10;
    }

我們可以將它展開為

    int[] x = new int[5];
    int[0] = 10;
    int[1] = 10;
    int[2] = 10;
    int[3] = 10;
    int[4] = 10;
 
省掉了迴圈結構後的執行效率會比較好．


#### 移除迴圈內不必要的程式碼

有些運算，在目前的程式碼中是被放在迴圈內的，但運算的本身與迴圈內的變數值並無關聯性，也就是可以移到迴圈外也不會影響整個結果的．例如：　

{% highlight java %}

int x = 10;
int y = 20;
int[] z = new int[10];
for(int i=0; i<10; i++) {
    z[i] = x+y;
}
{% endhighlight %} 

可改為:

{% highlight java %}
int x = 10;
int y = 20;
int r = x+y;
int[] z = new int[10];
for(int i=0; i<10; i++) {
    z[i] = r;
}
{% endhighlight %} 

少了在迴圈中的運算，效率也就提升了．

#### 最小化變數的作用範圍

最小化變數作用範圍的好處是增加程式碼的可讀性及可維護性，同時也能減少開發過程中的出錯機率。

在實作上，實現最小化變數作用範圍的方法就是在變數第一次被使用前才進行宣告。

我看過很多的原始碼寫法是在程式一開始時就把所有會用到的變數都宣告好，

但，這麼做只會造成混亂。在一開始看見變數被宣告時，我們還看不到它是如何被使用的，也就很難去判斷這樣的變數宣告是好還是不好；繼續往下閱讀程式碼，等到看見變數被使用的地方時，我們也忘了它在一開始是怎樣被宣告的，或是往宣告的地方到正在閱讀的程式碼，這中間是不是有經過別的處理。從程式碼的易讀性來看，這樣的程式碼至少在這方面實在是不夠好。

過早宣告變數，不僅使得作用域太早開始，也會造成作用域太晚結束。因為區域變數的作用範圍是從宣告點到環繞區塊結束為止。例如：

    if(true) {
        int x = 1+2+3;
    }

x的作用範圍只在區塊內，區塊外是無法取用x的。

若提早宣告x

    int x = 0;
    if(true) {
        x = 1+2+3;
    }

在區塊外依然可以取得x，　這有可能會導致一些錯誤的發生。
 
有個例外的情況是 try catch 區塊，如果在變數的初始化過程中有可能拋出 exception，那麼這個變數就應該在try catch區塊中被初始化，另外還有一個例外的情況是迴圈，一些大型物件放在迴圈中被重新建立好幾次並不是個很好的寫法．


### 更有效率的陣列

#### 優先考慮使用 Arrays

當我們需要使用陣列時，在Java中有三種選擇: array、ArrayList、Vector

許多程式設計師會選擇使用Vector，既簡單又易用。事實上，使用Array要比使用Vector快得多，造成這二者效率上差別的幾個原因是：

1. Vector 的 get() 進行了同步化處理 (synchronized)．
2. 使用 Vector 時的所有操作都使用函式呼叫．

以上的二個特性在 array 都不存在，也就快上許多。更何況，Vector 是使用 Array 實作出來的，建立一個Vectory，就建立了一個 Array，及其他操作的程式碼，因此，建立一個 Vector的成本，也比建立一個 Array高。

另一個選擇是 ArrayList，它其實是拿掉了同步化處理的Vector，在效率上肯定是比Vector好．但，array還是比ArrayList快．

至於，什麼時候才要使用Vector或ArrayList呢？

在我們無法確定有多少個元素會被添加到容器內時，就是使用Vector 或 ArrayList的時刻。因為Vectory或ArrayList都有自動調整本身大小的能力，雖然這能力要付出相當的代價來換取，Vectory或ArrayList每一次調整大小，是透過建立一個新的較大的array，再將所有元素從原本的容器中複製過去．

無法確定有多少個元素需要儲存的情況下，還有另外一種作法，就是建了一個足夠大的array，雖然可能多浪費了一些記憶體，但與使用Vectory或ArrayList比起來，在效益上可能是更好的．

### 異常處理

#### 別把異常"吞"了

所謂的"吞"，指的是：補捉異常， 但不做任何處理．

像這樣：

    public void test1() { 
        try { 
            test2(); 
        } catch (Exception e) { 
            // 沒有任何處理
        }  
    } 
 
    public void test2() throws Exception{ 
        throw new Exception("test2"); 
    } 

就好像什麼事都沒發生過，但如果這部份的Exception，在之後導致了其它的問題，將會增加除錯的困難度，因此，至少要將異常log下來,這部份可以使用log4j或是commons Logging機制．

 
#### 使用finally

這部份最常見到的地方應該是在使用JDBC對資料庫操作時, 用來在SQL出錯時將資料庫連線關閉.

如下:

    Connection conn = null; 
    
    try { 
        conn = DriverManager.getConnection(); 
        . 
        . 
        . 
    } catch (Exception e) { 
        logger.error(e.getMessage()); 
    } finally { 
        if (conn != null) { 
            try { 
                conn.close(); 
                conn = null; 
            } catch (SQLException e) { 
                logger.error(e.getMessage()); 
            } 
        } 
    } 

#### 不要在try或finally區塊中return

以下的程式碼回傳的是 1 還是 2 ? :
 
    public void getTest() { 
        try { 
            return 1; 
        } finally { 
            return 2; 
        } 
    } 

答案是2.
因為 finally 是一定會被執行的區塊.為了減少程式發生錯誤的機會, 應避免在 try 及 finally 中,使用 return.
