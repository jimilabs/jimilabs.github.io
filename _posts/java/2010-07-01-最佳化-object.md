### 物件的初使化

#### 不需給定初始值

在字串的部份, 常看到有人這樣寫:

    String x = "";
    x = "hello";

事實上, String型態的變數,每次給定新的值之後, 都是一個新的物件,因此一開始時給定的空字串等於是一個建立後,從來沒被使用過, 就立刻等著被銷毀的物件, 佔用了系統的資源, 郤又沒什麼用途.

因此, 比較好的寫法應該是在使用前才產生,產生之後就當常數來用, 如:

    String x = "hello";

另外還有一種字串的建立方式也是要避免的:

    String x = new String("hello");

在這一個例子中, 其實已經建立了二個字串的實體,

一個是x ,另一個是"hello", 也是一樣的多餘了

 

#### 減少建立物件的成本

一個物件, 內含的程式碼越多的話, 建立起來也會耗用較多的資源，

在建立一個物件時，會為物件內的所有變數及這個物件本身及其super class配置記憶體，

接著配置變數初始化的預設值．

由此可知，建立一個較輕量的物件，要比建立一個重型的物件快很多，因此在實務上建議可以以輕量的物件來取代重型物件．

### 物件的結束

Java提供了回收機制, 負責把不再被使用的記憶體回收，因此Java程式設計師往往就忽略了記憶體的問題，實是上，我們可以協助回收機制取回記憶體,

幫忙的做法也很簡單, 當一個物件不再被使用時, 就把它設為 **null**.
