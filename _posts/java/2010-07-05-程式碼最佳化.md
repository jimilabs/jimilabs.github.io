刪除空白的函式

有些工具能為我們產生空白的函式，再由程式設計師填入函式內的實際作用程式碼，

不過，在實際開發時才會發現並不是每個函式都用得到，因此就會留下一些空的函式．

這些空的函式，在最後編譯出的bytecode中，還是會出現，也會被呼叫．

但他們實際上並沒有用途，因此我們可以放心移除它們，以換取更高效能的程式碼．

 

刪除無用的程式碼

一份程式碼，在多個程式設計師的修改後，或許是邏輯改變了，也可能是因為粗心大意，而產生一些永遠不會被執行到的程式碼．

刪除這些程式碼可以讓程式更易讀，更快速．

 

展開迴圈

有時候，使用迴圈並不是必要的，如底下的這個例子：

{% highlight java %}

int[] x = new int[5];

for (int i=0; i<5; i++) {
    x[i] = 10;
}

{% endhighlight %} 

我們可以將它展開為

{% highlight java %}

int[] x = new int[5];

int[0] = 10;

int[1] = 10;

int[2]= 10;

int[3] = 10;

int[4] =10;

{% endhighlight %} 
 

省掉了迴圈結構後，這段程式碼的執行效率也更好了．

移除迴圈內不必要的程式碼

有些運算，在目前的程式碼中是被放在迴圈內的，但運算的本身與迴圈內的變數值並無關聯性，

也就是可以移到迴圈外也不會影響整個結果的．例如：　

{% highlight java %}

int x = 10;

int y = 20;

int[] z = new int[10];

for(int i=0; i<10; i++) {

z[i] = x+y;

}
{% endhighlight %} 

可改為:

{% highlight java %}

int x = 10;

int y = 20;

int r = x+y;

int[] z = new int[10];

for(int i=0; i<10; i++) {

z[i] = r;

}
{% endhighlight %} 

少了在迴圈中的運算，效率也就提升了．
