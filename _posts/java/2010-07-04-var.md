最小化變數作用範圍的好處是增加程式碼的可讀性及可維護性，同時也能減少開發過程中的出錯機率。

在實作上，實現最小化變數作用範圍的方法就是在變數第一次被使用前才進行宣告。

我看過很多的原始碼寫法是在程式一開始時就把所有會用到的變數都宣告好，

但，這麼做只會造成混亂。在一開始看見變數被宣告時，我們還看不到它是如何被使用的，也就很難去判斷這樣的變數宣告是好還是不好；繼續往下閱讀程式碼，等到看見變數被使用的地方時，我們也忘了它在一開始是怎樣被宣告的，或是往宣告的地方到正在閱讀的程式碼，這中間是不是有經過別的處理。從程式碼的易讀性來看，這樣的程式碼至少在這方面實在是不夠好。

過早宣告變數，不僅使得作用域太早開始，也會造成作用域太晚結束。因為區域變數的作用範圍是從宣告點到環繞區塊結束為止。例如：

if(true) {

int x = 1+2+3;

}

x的作用範圍只在區塊內，區塊外是無法取用x的。

若提早宣告x

int x = 0;

if(true) {

x = 1+2+3;

}

在區塊外依然可以取得x，　這有可能會導致一些錯誤的發生。

 

有個例外的情況是try catch區塊，如果在變數的初始化過程中有可能拋出exception，

那麼這個變數就應該在try catch區塊中被初始化，


另外還有一個例外的情況是迴圈，一些大型物件放在迴圈中被重新建立好幾次並不是個很好的寫法．
