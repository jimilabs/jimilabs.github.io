前言
----


早期的軟體多是採行client-server架構，使用圖形使用者界面的開發環境以及事件驅動模型，先稍微說明一下這兩個名詞。

**圖形使用者界面** --- Graphical User Interface ( GUI )，指的是使用採用圖形方式顯示的軟體操作界面，大家平常慣用的作業系統 Windows 以及在Windows上運作的多數軟體，都是採用GUI的操作方式，相對於GUI的另一種操作方法，則是命令列介面，使用指令的操作方式。

**事件驅動模型** --- event-driven model，這是相對於批次程式而言的設計方法，透過接收器(例如: 按鈕)，監聽器以及事件處理程式來完成一個動作。

近年來，Java大量應用於開發網站應用系統，但由於HTTP的stateless本質，使得開發人員必須親自處理一些傳統GUI能自動處理掉的細節。於是，有些open source計畫開始模擬傳統的GUI架構，例如﹕Tapestry，之後各家公司浱代表成立了Java Community Process (JCP) 團隊。在2004年三月首次發表了合作成果 – JavaServer Faces 1.0，現在的版本編號則已來到了2.0。

JavaServer Faces (JSF) 是一套定義UI元件的標準 ，雖然所有定義的UI元件都能應用在網頁裡，但是JSF並不限定只能使用在網頁上，實際會傳送什麼到使用者的界面是由繪製器(renderer)所決定的。

使用JSF主要的好處在於能發展出更模組化的UI元件。以目前最流行的MVC模型來說，可以很乾淨的將Model、View、Controller三個角色分開來並且很靈活的組合運用，但是很難把某一個動作所需要的Model、View、Controller包裝成一個元件，然後在不同專案中重覆利用。而JSF正好可滿足這方面的需求，後續的文章將有更詳盡的說明。


開發設定
--------

### 環境

要能執行JSF的系統，需要有二樣東西，一樣是 Java Web Container，另一樣是JSF的實作成品。

Java Web Container 最為人所熟知的應該是Tomcat，可由Apache Tomcat的網站取得﹕

* Apache Tomcat 官網 ﹕http://tomcat.apache.org/
* 直接下載 Tomcat  ﹕Tomcat 6.0.29

JSF的實作成品可由Mojarra專案取得

* Mojarra Project 官網﹕https://javaserverfaces.dev.java.net/
* 直接下載﹕2.0.3 binary bundle

下載回來的是一個壓縮檔，解開後在裡面找出二個檔案﹕jsf-api.jar、jsf-impl.jar，將二個檔案放在專案的 WEB-INF\lib 資料匣中。

#### 設定

在開發階段，JSF最基本的要在web.xml裡加入以下的片段﹕

##### javax.faces.STATE_SAVING_METHOD

     <context-param> 
        <param-name>javax.faces.STATE_SAVING_METHOD</param-name> 
        <param-value>client</param-value> 
    </context-param>

這段的設定用來指定JSF元件的狀態保存的位置，可使用的參數值有二個﹕

* client: 保存在客戶端。
* server: 保存在伺服器端。

預設值是server。設置在client可以減少一些不必要的麻煩。

##### javax.faces.PROJECT_STAGE

    <context-param> 
        <param-name>javax.faces.PROJECT_STAGE</param-name> 
        <param-value>Development</param-value> 
    </context-param>

這邊可以設定JSF專案的狀態，在程式裡能透過 Application.getProjectStage()取得設定的值，可使用的值為﹕

* Production
* Development
* UnitTest
* SystemTest
* Extension

預設為Production，這邊設置為Development會有助於解決開發過程中遇到的問題。

##### javax.faces.webapp.FacesServlet
 
    <servlet> 
        <servlet-name>Faces Servlet</servlet-name> 
        <servlet-class>javax.faces.webapp.FacesServlet</servlet-class> 
    </servlet>
    <servlet-mapping> 
         <servlet-name>Faces Servlet</servlet-name> 
         <url-pattern>*.faces</url-pattern> 
    </servlet-mapping>

這部份的設定就是用來指定觸發JSF的方式了。
 
有了必要的jar檔，也做好設定後，專案就有了執行JSF的能力了。
