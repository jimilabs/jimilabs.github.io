簡單的說, Ant是一套以Java為基礎的建置工具.
什麼是建置工具?

我們在開發環境寫好程式之後， 需要將程式發佈到真正的執行環境中，這個發佈的過程會有些例行的工作要做，例如，編譯原始碼、複製檔案、修改設定檔…，而建置工具就是用來執行這些例行工作的工具 。

ANT 就是一套這樣的建置工具，簡單易用，並且跨平台，擴充性及彈性都很好，它能建置、測試、部署各種專案。

## ANT的主要概念

了解ANT的主要概念，才能了解ANT

**建置檔**

建置檔是用來描述ANT如何建置、測試、部署系統的檔案，使用的是XML格式。


**專案**

每一個建置檔都包含了一個專案。

**目標**

在建置檔的一個專案中，會包含多個目標(target)，這些目標可能是專案建置過程中的實際輸出、或是建置過程的某個階段。一個目標可依附一個或多個其它的目標，所謂的依附指的是在執行這個目標前，所要先執行的目標。

**任務**

在上述的”目標”中, 會由一到多個任務(task)來達成這個目標，每一個任務其實就是一個ANT的程式庫中的JAVA類別，能用來完成各程不同的工作。
專案範例

一個典型的建置檔 build.xml

{% highlight xml %}

<?xml version="1.0" ?>
<project name ="FirstProject" default="deploy">

    <target name="init" >
        <mkdir dir="dist">
        <mkdir dir="dist/classes">
    </target>

    <target name="compile" depends="init">
        <javac srcdir="src" destdir="dist/classes" />
    </target>
    
    <target name="deploy" depends="compile">
        <jar destfile="dist/FirstProject.jar" basedir="dist/classes" />
    </target>

</project>

{% endhighlight %} 

在這個例子裡， ANT建立了二個目錄，再將原始碼編譯後壓縮成一個jar檔。這是一個簡單的例子，但由這個例子應該不難看出上述所提的核心概念: XML格式、一個專案(project)，多個目標(init、compile、deploy)，任務(mkdir、javac、jar)。
