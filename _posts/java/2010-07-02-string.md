字串連接有幾種做法

方法一：

String x = "hello" + " world";

 

方法二：

StringBuffer x = new StringBuffer();

x.append("hello");

x.append(" world");

 

方法三：

StringBuilder x = new StringBuilder();

x.append("hello");

x.append(" world");

 

以上的三種方法哪種好?

 

答案是：　第三種方法最好．

 

使用String物件直接做字串連接的第一種方法是效率最差，也最佔資源的一種做法，

因為String物件是不可變的，因此每一次連接了一個新字串後，都是重新建立一個新的物件，

效能也就很難好得起來．

 

使用StringBuffer做字串連接是比較好的做法，在效能上，比使用String要快上許多，

若使用JDK 5進行開發的話，使用StringBuilder又會比使用StringBuffer快，

二者的差別是StringBuffer在多執行緒下，是安全的，因為StringBuffer進行了synchronized處理，

但也就是因為synchronized，讓StringBuffer效能不如StringBuilder，

而多數情況下，我們可以使用許多做法來避免需要多執行緒安全的情況，

因此在多數情況下，都是單執行緒的，也就是能使用StringBuilder

這會是目前最好的字串連接做法．
