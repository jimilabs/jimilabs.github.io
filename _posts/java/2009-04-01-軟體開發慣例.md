不同的人，不同的編輯器，定義的tab大小可能都不一樣，為了使格式一致，方便大家閱讀程式碼，建議使用四格空白取代tab

### SQL

#### SQL 大小寫

1.  SQL標準語法使用大寫
2.  Table 或 Column 使用小寫
    * MySQL在Linux中，table名稱的大小寫是不同的，預設是大寫；而在Windows中，table名稱的大小寫是相同的，預設是小寫。因此，為了相容性與移植性，在建立Table時，一律使用小寫。

3.
  條件式中的運算子置於左方

    SELECT col1, col2
       FROM  tb1 t1
      WHERE col1 > 1
        AND col2 > 2;

    部份SQL開發工具即提供了SQL格式化的功能，例如﹕TOAD、PL/SQL


#### 明確寫出欄位名稱

在SQL裡，應盡量避免使用星號，而將需要的欄位明確出效能才會好

這樣的寫法﹕

    SELECT col1, col2 FROM tb1;

效能會好過﹕

    SELECT * FROM tb1;

#### 使用別名, 從一個table以上取出資料時

若未使用別名，資料庫需要去解析各別欄位分屬於哪一個Table

比較不好的寫法

    SELECT col1,col2,cola,colb FROM tb1, tb2 WHERE col3 = colc;

比較好的寫法

    SELECT a.col1,a.col2,b.cola,b.colb FROM tb1 a, tb2 b WHERE a.col3 = b.colc;

#### 集中條件在單一Table上

在join多個table時, 盡量將條件限制在一個table，如此就能大幅減少這一個Table的回傳筆數, 也就減少了整段SQL所需的運算。

#### in 條件式

使用 in 條件式比對的 sub query 應避免對欄位再加工

例如:

    SELECT col1,col2
      FROM tb1
     WHERE col3 IN (SELECT CONCAT(b.col1,b.col2) FROM tb2 b )

#### 其他改善的方法


* 優先調整SQL Statement
* 增加Index,但要注意的是，增加Index雖然可以提升select的效能，但郤可能降低insert、update、delete的效能
* Cache 經常使用的小型 Tabls 或 Index
* Join A Table的目的是篩選資料而非擷取欄位時，將 A Table 到 Sub-quey用 EXISTS來替代。
