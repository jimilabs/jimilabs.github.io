JSTL(The JavaServer Pages Standard Tag Library) 是目前開發網站應用程式時，一組廣為使用的標籤庫，這組標籤庫中，也提供了制做多國語系網站所需要的i18n功能。

## 第一步：建立語系檔

假設，我們要製作的是繁體中文語系、英文語系，

先在\WEB-INF\classes底下放置二個檔案：　

MessageResources.txt
MessageResources_en.txt
這兩個檔案在稍後，會分別轉成MessageResources.properties及MessageResources_en.properties。

MessageResources.properties裡頭放的，是繁體中文語系，也是網站預設的語系，

另一個MessageResources_en.properties放的，當然就是英文語系了。

 

## 第二步：編輯語系檔

以"歡迎"這兩個字為例，

在MessageResources.txt裡頭寫上

{% highlight ruby %}
welcome = 歡迎
{% endhighlight %}

在 MessageResources_en.txt 裡頭寫上

{% highlight ruby %}
welcome = Welcome
{% endhighlight %}

 

## 第三步：native2ascii 轉碼

同樣的，語系檔需要轉碼過才能正常顯示，

同時在這邊，我們要將檔案名稱轉為正確的檔案名稱。

語法如下：

{% highlight ruby %}
native2ascii.exe MessageResources.txt MessageResources.properties

native2ascii.exe MessageResources_en.txt MessageResources_en.properties
{% endhighlight %}

 

第四步：指定語系

在JSP的頁面上，要先宣告使用JSTL的fmt標籤

{% highlight ruby %}
<%@ taglib prefix="fmt" uri="http://java.sun.com/jstl/fmt" %>
{% endhighlight %}

 

接著，我們根據使用者瀏覽器的語系，來設定要顯示的語系檔

{% highlight ruby %}
<fmt:setLocale value="${header['accept-language']}"/>

<fmt:setBundle basename="MessageResources"/>

{% endhighlight %}
 

如果要讓使用者自已選定語系的話，

只要修改第一行　fmt:setLocale　，讓使用者指定這個tag的值就行了。

第五步：取出資源

在JSP頁面上將值取出所應用的tag是　fmt:message，

完整語法為　

{% highlight ruby %}
<fmt:message key="welcome" />
{% endhighlight %}

如此，便完成了
