在處理異常時，有幾個需要特別注意的地方：

### 別把異常"吞"了


所謂的"吞"，指的是：補捉異常， 但不做任何處理．

像這樣：

{% highlight java %}

public void test1() { 
    try { 
        test2(); 
    } catch (Exception e) { 
    }  
} 
 
public void test2() throws Exception{ 
    throw new Exception("test2"); 
} 

{% endhighlight %}

就好像什麼事都沒發生過，但如果這部份的Exception，在之後導致了其它的問題，將會增加除錯的困難度，因此，至少要將異常log下來,這部份可以使用log4j或是commons Logging機制．

 
### 使用finally


這部份最常見到的地方應該是在使用JDBC對資料庫操作時, 用來在SQL出錯時將資料庫連線關閉.

如下:

{% highlight java %}
Connection conn = null; 
try { 
    conn = DriverManager.getConnection(); 
    . 
    . 
    . 
} catch (Exception e) { 
    logger.error(e.getMessage()); 
} finally { 
    if (conn != null) { 
        try { 
            conn.close(); 
            conn = null; 
        } catch (SQLException e) { 
            logger.error(e.getMessage()); 
        } 
    } 
} 
{% endhighlight %}

<br> 

### 不要在try或finally區塊中return

想一下以下的程式碼:

 
{% highlight java %}
public void getTest() { 
    try { 
        return 1; 
    } finally { 
        return 2; 
    } 
} 
{% endhighlight %}

在呼叫 getTest 時,回傳的是 1 還是 2 ? 

答案是2.

因為 finally 是一定會被執行的區塊.

為了減少程式發生錯誤的機會, 應避免在 try 及 finally 中,使用 return.
