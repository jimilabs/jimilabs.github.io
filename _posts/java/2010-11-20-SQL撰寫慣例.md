好的SQL撰寫慣例，不僅有助於閱讀，對執行效能也有很大的影響
SQL大小寫慣例

SQL標準語法使用大寫
Table 或 Column 使用小寫
條件式中的運算子置於左方
example:
SELECT col1, col2
    FROM  tb1 t1
    WHERE col1&gt;1
    AND   col2&gt;2;

部份SQL開發工具即提供了SQL格式化的功能，例如﹕TOAD、PL/SQL


明確寫出欄位名稱

在SQL裡，應盡量避免使用星號，而將需要的欄位明確出效能才會好
這樣的寫法﹕
SELECT col1, col2 FROM tb1;

效能會好過﹕
SELECT * FROM tb1;

使用別名, 從一個table以上取出資料時

若未使用別名，資料庫需要去解析各別欄位分屬於哪一個Table

比較不好的寫法
SELECT col1,col2,cola,colb FROM tb1, tb2 WHERE col3 = colc;

比較好的寫法
SELECT a.col1,a.col2,b.cola,b.colb FROM tb1 a, tb2 b WHERE a.col3 = b.colc;

集中條件在單一Table上


在join多個table時, 盡量將條件限制在一個table，如此就能大幅減少這一個Table的回傳筆數, 也就減少了整段SQL所需的運算。

in 條件式

使用in條件式比對的sub query應避免對欄位再加工

例如:

SELECT col1,col2
FROM tb1
WHERE col3 IN (SELECT CONCAT(b.col1,b.col2) FROM tb2 b )


其他改善的方法


優先調整SQL Statement
增加Index,但要注意的是，增加Index雖然可以提升select的效能，但郤可能降低insert、update、delete的效能
Cache經常使用的小型Tables或Indexes
Join A Table的目的是篩選資料而非擷取欄位時，可將A Table 移到Sub-query用EXISTS來替代。



後記

對於一個資料庫應用系統來說，資料庫的設計、SQL的寫法，是整個系統效能的關鍵，在設計實作時就能多一點心的話，對於後期效能的調整或是日後系統上線後的使用者感受，都有很大的幫助。
