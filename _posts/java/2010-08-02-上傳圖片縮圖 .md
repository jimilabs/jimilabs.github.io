最新消息這功能，在後台會使用html編輯器上稿，在大部份的案子中，在html編輯器裡提供插入圖片的功能是必要的，但會衍生出另一個的問題，就是插入或上傳的圖片太大，造成前台在顯示時,版面被破壞，這個問題,有二個解決方法

請使用者縮圖後再上傳。
由程式將使用者上傳的圖片縮小。
 
相信所有使用者的選擇都會是選項2，因此接下來要介紹的就是由程式縮圖的方法。

這邊使用的是JAI (Java Advanced Imaging)來進行縮圖，實際上的程式碼寫法，可參考下面的例子，而這個涵式的使用時間，就是在圖片被上傳後，可先經由這個函式將圖片縮小，再進行儲存，這樣一來，就能避免版面被圖片撐開的情況了。

{% highlight java %}
            
public static void resizeImg(InputStream inputStream, File newFile, float newWidth, float newHeight) {
  
      SeekableStream seekStream = SeekableStream.wrapInputStream(inputStream, true);
      RenderedOp image = JAI.create("stream", seekStream);
      ((OpImage) image.getRendering()).setTileCache(null);

      int oldWidth = image.getWidth();
      int oldHeight = image.getHeight();
    
      float scale = 0;
      float scaleW = newWidth / oldWidth;
      float scaleH = newHeight / oldHeight;
    
      float proportionH = 0;
      float proportionW = 0;
      if (newWidth >= oldWidth && newHeight >= oldHeight) {
          scale = 1;
      } else {
          proportionW = newWidth / oldWidth;
          proportionH = newHeight / oldHeight;
    
          if (proportionW > proportionH) {
              scale = proportionH;
          } else {
              scale = proportionW;
          }
      }
    
      ParameterBlock pb = new ParameterBlock();
      pb.addSource(image); 
      pb.add(scale); // The xScale
      pb.add(scale); // The yScale
      pb.add(0.0F); // The x translation
      pb.add(0.0F); // The y translation
      pb.add(new InterpolationNearest()); // The interpolation

      RenderedOp resizedImage = JAI.create("scale", pb, null);

      try {
          FileOutputStream outputStream = new FileOutputStream(newFile);
          JAI.create("encode", resizedImage, outputStream, "PNG", null);
          outputStream.close();
      } catch (Exception e) {
      }
  }
